Fall 2016 — Introduction to Software Design — Assignment 3

Group size: Two

Due date

The due date for A3 is extended to 5pm Sunday 27 November.

Learning Objectives

  • Interact with another programmer working on the same code
  • Work with version control
  • Write a GUI
  • Refactor existing code to apply design patterns
  • Write a set of unit tests (in addition to finding bugs, it's a a good way to help decide on preconditions for your methods)
  • Think about the implications of exceptions on the user experience

Overview

Using the same repository as for A2 (but in a new subdirectory), write the GUI for the photo renamer, write full Javadoc for your program, use proper Java style, and include unit tests for the parts of your program that manage the master list of tags. (You are encouraged to write more unit tests, both for practice and because it can help you think of corner cases.)

You must use at least two design patterns in your code (they can be from lecture or from your own research), and indicate (using comments in your code) which classes are involved in each pattern, and what their roles are.

We will mark your solution based on appropriate functionality, and also based on Code Smells and on your Java style.

You are strongly encouraged to refactor your A2 code to improve it.

Required code structure

These are vital:

  • the class containing the main method must be named PhotoRenamer
  • PhotoRenamer must be in package photo_renamer

  • The structure of your group repository must be this: 

    group_name/a3/src/photo_renamer

Notice that you will need to copy over any code from the group_name/a2 subdirectory.

You are welcome to have other packages in the src directory.

On the command line, this sequence of commands must run without error in CDF: 

git clone https://markus.teach.cs.toronto.edu/git/csc207-2016-09/group_name
cd group_name/a3/src
javac photo_renamer/PhotoRenamer.java
java photo_renamer.PhotoRenamer

We will continue to check whether both partners are fully participating in the program development and that both of you are properly using version control.

Partners
